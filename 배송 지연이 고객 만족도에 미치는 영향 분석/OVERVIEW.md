# 배송 지연 분석 프로젝트

## 1. 프로젝트 개요

본 프로젝트는 브라질 이커머스 플랫폼 'Olist'의 주문 데이터를 활용하여 배송 지연이 고객 만족도에 어떤 영향을 미치는지 분석한 프로젝트입니다.

주문 정보, 배송 일정, 고객 리뷰를 결합하여 배송 성과와 고객 반응 간의 관계를 수치화 하여 확인하고 운영 개선에 활용 가능한 인사이트를 도출할 예정입니다.

프로젝트 기간 : 2025년 12월 26일 ~ 현재

활용 데이터 셋 : [Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

활용 언어 : SQL, Python

시각화 툴 : Python, Tableau

## 2. 분석 배경과 문제 정의

Olist는 다수의 판매자가 참여하는 마켓플레이스 구조이며 배송 성과가 판매자와 지역에 따라 달라질 가능성이 큽니다. 

배송 지연이 지속적으로 발생하면 고객 만족도가 하락할 수 있고 이는 제품의 리뷰 점수 하락과 판매자, 배송 업체, 그리고 플랫폼 자체의 신회도 저하로 이어질 수 있습니다

그래서 이러한 질문을 통해 분석을 시작했습니다.

- 배송 지연이 고객의 리뷰 점수에 부정적인 영향을 미치는가?
- 배송 지연이 특정 지연에서 더 빈번하게 발생을 하는가?

## 3. 데이터 소개

사용한 데이터셋은 위에 링크를 첨부한 Kaggle의 Olist 데이터셋이고,해당 데이터 셋은 2016년부터 2018년까지의 약 10만건의 주문 정보를 포함하고 있습니다.

테이블은 다음과 같습니다. 
- olist_customers_dataset
- olist_geolocation_dataset
- olist_order_items_dataset
- olist_order_payments_dataset
- olist_order_reviews_dataset
- olist_orders_dataset
- olist_products_dataset
- olist_sellers_dataset
- product_category_name_translation

테이블의 스키마에 대해서는 "data" file을 참고 부탁드립니다.

## 4. 분석 질문 정의

배송 지연 정의는 다음과 같이 설정해놓고 시작하겠습니다. 

- 지연 주문 : 실제 배송 완료일이 예상 배송일보다 늦은 주문
- 정상 주문 : 실제 배송 완료일이 예상 배송일보다 빠르거나 같은 주문

이에 따라 검증해야 할 질문은 다음과 같이 설정했습니다. 

- 지연 주문과 정상 주문과의 리뷰 점수 평균이 다른가?
- 판매자/구매자 지역별 배송 지연 비율은 다른가?

## 5. 데이터 전처리

- 주문과 리뷰 데이터는 order_id 기준으로 JOIN
- 예상 배송일, 실제 배송일 등 배송 날짜 관련 컬럼들의 datatype은 모두 yyyy-mm-dd로 통일
- 실제 배송 완료일과 예상 배송일이 모두 존재하는 주문만 포함
- 리뷰 점수가 없는 주문은 만족도 분석에서 제외
- 배송 지연 여부 (boolean) 컬럼을 생성

## 6. 분석 방법

- 배송 지연 여부에 따른 리뷰 점수 분포 비교
- 평균 리뷰 점수 차이를 계산
- 지역별 배송 지연 비율을 계산 및 비교

**해당 프로젝트는 위의 project scope대로 진행할 예정이며, result 파일에 분석 결과 및 SQL 쿼리, 인사이트, 한계와 배운점 등을 서술할 예정입니다.**
